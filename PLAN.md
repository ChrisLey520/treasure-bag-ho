## 二、产品设计

### 视觉风格（"一定要好看"）

- **首页**：顶部大标题 + 搜索框，分类 Tab（推荐/图片/文字/生活/查询/开发者），2 列渐变圆角卡片宫格，每个工具独立渐变色 + 图标
- **主题**：跟随系统深浅色，自定义主题色板（AppStorage + @StorageLink）
- **动效**：卡片按压缩放（`.scale` + spring 动画）、页面转场（Navigation 自定义转场）
- **字体层级**：大标题 28fp / 卡片标题 16fp / 说明 12fp，留白充足
- **收藏**：长按卡片收藏，首页"我的常用"置顶

### 技术架构

```
treasure-bag/
├── AppScope/                    # bundleName: com.chris.treasurebag
├── entry/src/main/ets/
│   ├── entryability/EntryAbility.ets
│   ├── pages/Index.ets          # Navigation 根页面（首页宫格）
│   ├── pages/tools/…            # 每个工具一个页面
│   ├── model/ToolRegistry.ets   # 工具注册表（id/名称/图标/分类/渐变色/路由）
│   ├── components/ToolCard.ets  # 渐变工具卡片
│   └── utils/                   # 纯逻辑（换算、颜色、农历等）
└── build-profile.json5
```

- 路由：`Navigation` + `NavPathStack`，工具注册表驱动（新增工具只需注册一条记录）
- 状态：@State/@Prop 单向数据流；收藏与设置用 `@ohos.data.preferences`
- 列表：宫格 `Grid` + `LazyForEach`
- 遵循 API 24 规范：`@kit.*` 导入、`getMainWindowSync()`、`setSpecificSystemBarEnabled` 等（详见 harmonyos skill）

## 四、构建方式（命令行，不依赖打开 DevEco）

```bash
export DEVECO=/Applications/DevEco-Studio.app/Contents
export JAVA_HOME=$DEVECO/jbr/Contents/Home           # PackageHap 需要 Java
export PATH=$JAVA_HOME/bin:$DEVECO/tools/node/bin:$PATH
export DEVECO_SDK_HOME=$DEVECO/sdk                    # HarmonyOS 6.1.1 / API 24
node $DEVECO/tools/hvigor/bin/hvigorw.js --mode module -p product=default assembleHap --no-daemon
```

产物：`entry/build/default/outputs/default/entry-default-unsigned.hap`（未签名，真机安装需在 DevEco 中配置签名）。

## 五、风险与说明

1. **平台限制**：自动点击、清理加速等在 HarmonyOS 沙箱下无法实现，已标 ❌
2. **网络 API 选型**：Phase 5 前需要确定免费 API 源（如 60s API、和风天气免费档）
3. **上架**：需华为开发者账号签名；本计划先保证 DevEco 本地可编译运行
